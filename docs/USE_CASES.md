# gws-harness 페르소나별 사용 가이드

> Google Workspace(Gmail + Calendar)와 Claude Code를 연결해서 매일 반복되는 메일 정리·일정 계획·주간 회고를 대화형으로 처리하는 플러그인입니다. 2계정(회사 + 개인) 지원 버전(v1.1.0)부터는 두 계정을 한 번에 조회할 수 있습니다.

구체적인 하루/한 주에 어떤 대화가 오가는지는 [SCENARIOS.md](SCENARIOS.md)에서 볼 수 있습니다.

---

## 1. 회사 + 개인 2계정을 동시에 보고 싶은 사람

**이 사람은 누구인가**

회사 이메일과 개인 이메일을 둘 다 쓰는 직장인. 업무시간에는 회사 메일이 급하고, 퇴근 후엔 개인 메일이 중심인데, 매번 브라우저 탭을 전환하기 귀찮은 사람.

**액션 플랜**

1. `/gwh:credential-init work` — 회사 계정 등록
2. `/gwh:credential-init personal` — 개인 계정 등록
3. 아침에 `/gwh:triage` — 두 계정 미읽음 메일을 한 번에 분류
4. `/gwh:brief` — 오늘 일정 + 메일 요약을 한 화면으로 확인

**아웃풋**

- `~/.gwh/triage-2026-04-19.md` — 회사/개인 메일이 한 파일에 🏢/👤 아이콘으로 구분되어 긴급도 순으로 정렬
- 업무시간 자동 감지: 평일 09-18시엔 회사 메일이 더 눈에 띄게 표시, 저녁/주말엔 개인 메일이 강조
- 계정별 개별 파일도 함께 생성 (`triage-work-2026-04-19.md` 등) — 한 계정만 따로 보고 싶을 때 사용

## 2. 매일 아침 5분 브리핑이 필요한 사람

**이 사람은 누구인가**

출근하자마자 "오늘 뭐부터 할지" 정리하고 싶은 실무자. 미팅·메일·업무가 섞여 있어 우선순위 판단이 필요한데 직접 정리할 시간이 없는 사람.

**액션 플랜**

1. `/gwh:triage` — 미읽음 메일 분류 (긴급/액션/FYI/무시 4단계)
2. `/gwh:brief` — 캘린더 일정 + 메일 요약 + Jira 스프린트 상태를 하나로 통합
3. 필요 시 특정 메일만 골라 `/gwh:mail-to-ticket` — Jira 이슈 또는 Notion 페이지로 변환

**아웃풋**

- `~/.gwh/brief-2026-04-19.md` — "오늘 긴급 2건 / 미팅 3개 / 집중 가능 3시간" 형식의 한 장 요약
- 메일은 Gmail에 있고, 일정은 Calendar에 있고, 티켓은 Jira에 있는 상황에서 각 시스템을 따로 열지 않아도 됨
- 같은 날 두 번째 실행하면 캐시 재사용으로 4밀리초 만에 결과 표시

## 3. Jira 티켓을 캘린더에 시간 블록으로 배치하고 싶은 사람

**이 사람은 누구인가**

스프린트 시작일에 "이 티켓들 언제 할지" 캘린더에 미리 박아두고 싶은 개발자/기획자. 빈 시간이 어디 있는지 일일이 확인하기 번거로운 사람.

**액션 플랜**

1. `/gwh:cal-plan --account work` — Jira 스프린트 티켓 자동 조회 + 이번 주 캘린더 빈 시간 계산
2. 제안된 타임블록 배치안 확인 → 승인 또는 수정
3. 승인하면 Google Calendar에 이벤트 자동 생성 (스토리 포인트 기반 시간 산출: SP 1 = 1.5시간, SP 3 = 5시간 등)

**아웃풋**

- 캘린더에 실제 이벤트 생성: "[DPPM-91] hex 감사 (1/2)" 09:00-12:00 식으로
- `~/.gwh/cal-plan-work-2026-04-19.md` — 어떤 티켓을 언제 배치했는지 기록 (되돌아보기용)
- 빈 시간이 부족하면 "목/금 미팅 4개라 3시간만 남음 → 우선순위 조정 필요" 같은 경고

**주의**

이벤트 쓰기는 반드시 한 계정에만 들어가도록 `--account work` 또는 `--account personal` 명시 필요 (실수로 다른 계정에 이벤트가 추가되는 사고 방지).

## 4. 주간 회고 자료를 만들어야 하는 사람

**이 사람은 누구인가**

매주 금요일에 "이번 주 뭐 했지" 정리해야 하는 팀 리드 / PM / 1인 개발자. 미팅이 너무 많은지, 특정 사람과만 소통하고 있는 건 아닌지 객관적 숫자로 보고 싶은 사람.

**액션 플랜**

1. `/gwh:digest` — 이번 주 월요일부터 오늘까지 자동 집계
2. 결과를 회고/1:1 자료로 활용

**아웃풋**

- `~/.gwh/digest-2026-W16.md` — 아래 항목 자동 집계:
  - 이번 주 메일 송수신 건수 (계정별)
  - 가장 많이 소통한 사람 Top 3
  - 미팅 횟수 + 총 시간 + 업무시간 대비 비중 (30% 초과 시 "미팅 과부하" 경고)
  - 완료한 Jira 티켓 (SP 합계, 유형별 분류)
  - 코딩 가용 시간 추정: 40시간 - 미팅 시간 - 메일 작성 시간
- 전주 대비 증감도 표시 (미팅 ±N회, SP ±M)

## 5. 외부 사용자 / 오픈소스 사용자

**이 사람은 누구인가**

회사의 Jira·Notion 연동 없이 Gmail + Calendar만 쓰는 개인 사용자. Claude Code를 처음 써보거나 gws-harness를 공유받아서 시도하는 사람.

**액션 플랜**

1. Google Workspace CLI(`gws`) 설치 + 인증
2. Claude Code에서 `/plugin marketplace add jangwonyoon/gws-harness` → `/plugin install gws-harness@gws-harness`
3. `/gwh:credential-init <원하는-이름>` — 계정 1개만 등록해도 OK
4. `/gwh:triage` / `/gwh:brief` — Gmail + Calendar 기반 기능만 사용

**아웃풋**

- Jira 스프린트 섹션이나 Notion 페이지 생성 기능은 자동으로 숨겨짐 — 오류 나지 않음
- 단일 계정 사용 시 기존 단순 포맷 유지 (계정 아이콘/라벨 없음)
- 메일 분류, 아침 브리핑, 캘린더 조회 등 핵심 기능은 전부 동작

## 6. 새 스킬을 추가하거나 플러그인을 고치고 싶은 기여자

**이 사람은 누구인가**

본인 팀에 맞게 커스텀하려는 AI 엔지니어, 또는 PR로 기여하려는 개발자. 또는 Claude Code 플러그인을 처음 만들어보는 사람.

**액션 플랜**

1. 레포 fork → 클론
2. `AGENTS.md` 읽기 (기여자 가이드: 스킬 추가 절차, 네이밍 규칙, PR 워크플로우)
3. 2계정 관련 스킬을 만들 거면 `skills/shared/references/multi-account.md` 참조 (공통 패턴 6개 섹션 설명)
4. 새 스킬을 `skills/<name>/SKILL.md`로 추가, frontmatter 규격 준수
5. PR 생성 → CI(frontmatter 검증 + markdownlint) 통과 → 리뷰

**아웃풋**

- 본인만의 `/gwh:*` 스킬 (예: `/gwh:email-digest`, `/gwh:meeting-notes` 등)
- 기존 2계정 인프라(`~/.gwh/config.yml`, dual write, security 기본값) 그대로 활용 가능 — 다시 구현 불필요
- PR이 머지되면 플러그인 마켓플레이스를 통해 다른 사용자도 자동 업데이트

## 공통 산출물 위치

모든 결과물은 `~/.gwh/` 폴더에 마크다운으로 저장됩니다:

```
~/.gwh/
├── config.yml                           # 등록된 계정 정보 (자동 관리)
├── triage-2026-04-19.md                 # 메일 분류 (두 계정 통합)
├── triage-work-2026-04-19.md            # 회사 계정만
├── triage-personal-2026-04-19.md        # 개인 계정만
├── brief-2026-04-19.md                  # 아침 브리핑
├── cal-plan-work-2026-04-19.md          # 타임블록 배치 기록
├── ticket-2026-04-19-버그리포트.md      # 메일→티켓 변환 결과
└── digest-2026-W16.md                   # 주간 회고
```

- 이 폴더는 `chmod 0700` + Time Machine 제외가 자동 적용됩니다 (메일 본문 스니펫 포함이라 보안 처리).
- iCloud/Dropbox/OneDrive 하위에 두면 자동 차단됩니다 (동기화 차단, 로컬 경로 권장).
