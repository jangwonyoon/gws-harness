# gws-harness 실제 사용 시나리오

> 위 [USE_CASES.md](USE_CASES.md)가 "누가 어떻게 쓰는가"를 정리한 사전 같은 문서라면, 이 문서는 구체적인 하루/한 주에서 어떤 대화가 오가는지 보여주는 스토리 모음입니다.

## 시나리오 1: 월요일 아침 9시 10분 — 회사 메일과 개인 메일이 섞여 있는 아침

**상황**: 김실장은 회사 메일 계정(`@kakaohealthcare.com`)과 개인 Gmail을 둘 다 쓴다. 주말 동안 쌓인 메일이 회사 30통, 개인 20통. 어느 것부터 봐야 할지 감이 안 잡힌다.

**대화**

> 김실장: "오늘 메일 정리해줘"

Claude가 `/gwh:triage`를 실행. 약 4초 후 터미널에 결과가 뜬다.

```
📬 Gmail Triage — 2026-04-19 (통합)
work 약 30+통 / personal 약 20+통 분석 완료

🔴 긴급 (2건)
1. [🏢 work] [CEO] 월요일 오전 미팅 자료 재확인 요청
   → 액션: 자료 첨부 회신 | 마감: 오늘 10:30
2. [👤 personal] [은행] 카드 이상거래 확인 요청
   → 액션: 전화 확인

🟡 액션 필요 (5건)
1. [🏢 work] [기획팀] 스펙 리뷰 승인 대기 (DPPM-112)
...
```

**얻은 것**: 50통을 직접 열어볼 필요 없이, 지금 당장 처리할 2건이 뭔지 30초 만에 파악. CEO 메일을 먼저 답하고, 회의 끝난 뒤 은행에 전화하면 된다는 순서가 자동으로 잡힌다.

## 시나리오 2: 평일 오전 9시 — 오늘 뭐 할지 5분 안에 정하기

**상황**: 박개발자는 출근해서 커피 한 잔 내리는 동안 오늘 일정을 머릿속에 넣고 싶다. 매일 아침 15분씩 "오늘 뭐 하지" 고민하느라 몰입 시작이 늦는 게 고민.

**대화**

> 박개발자: "오늘 뭐해?"

Claude가 `/gwh:brief` 실행. 캐시가 있어서 4밀리초 만에 응답.

```
☀️ Morning Brief — 2026-04-19 (월)

📬 메일 (35통 미읽음)
- 🔴 긴급 1건: CEO 자료 회신
- 🟡 액션 3건: 스펙 리뷰, PR 승인, 1:1 일정 확인

📅 오늘 일정
| 시간 | 제목 | 참석자 |
|------|------|--------|
| 10:00-11:00 | 스프린트 플래닝 | 6명 |
| 14:00-14:30 | 김팀장 1:1 | 2명 |

빈 시간: 09-10시, 11-14시, 14:30-18시 (총 6.5h)

🏃 스프린트 (D-3)
완료 8/15 | 블로커 1건
내 In Progress: DPPM-91 (hex 감사)

💡 오늘 추천: 오전 빈 1시간에 CEO 회신, 점심 후 3시간은 DPPM-91 몰입
```

**얻은 것**: 5분 브리핑으로 오늘의 우선순위와 몰입 가능 시간 확보. "집중 3시간 있다"는 걸 알고 나면 미팅 사이 15분 단위로 조각내지 않고 큰 블록을 지킨다.

## 시나리오 3: 스프린트 시작 화요일 — 티켓을 캘린더에 시간 블록으로 꽂기

**상황**: 이시니어는 이번 스프린트에 티켓 5개를 할당받았다. 매번 "언제 할지" 막연히 생각만 하다 금요일에 허둥지둥 마감 치는 패턴이 반복되어, 미리 캘린더에 박아두고 싶다.

**대화**

> 이시니어: "이번 주 타임블록 짜줘 --account work"

Claude가 `/gwh:cal-plan --account work` 실행. Jira에서 티켓 5개 가져오고, 이번 주 work 캘린더 빈 시간 계산.

```
📅 타임블록 배치 제안 (work + personal 캘린더 union 기준)

### 화요일 (4/20)
| 시간 | 태스크 | 예상 |
|------|--------|------|
| 09:00-12:00 | DPPM-91 hex 감사 (1/2) | 3h |
| 14:00-17:00 | DPPM-91 hex 감사 (2/2) | 3h |

### 수요일 (4/21)
| 09:30-11:00 | DPPM-95 버그픽스 | 1.5h |
| 14:00-16:00 | DPPM-102 API 리뷰 | 2h |

⚠️ 목/금은 미팅 4개로 가용시간 3h만 남음

> 16개 이벤트를 work 캘린더에 생성할까요? [승인 / 수정 / 취소]
```

> 이시니어: "승인"

Claude가 work 캘린더에 이벤트 16개 생성. personal 캘린더는 건드리지 않음.

**얻은 것**: 주초에 한 번 플래닝하면 목요일에 "아 금요일 전엔 못 끝내겠다" 허둥대는 패턴 해소. 주말 개인 일정과 충돌 없이 work 캘린더에만 깨끗하게 박힘.

## 시나리오 4: 금요일 오후 5시 — 주간 회고 10분 컷

**상황**: 정팀장은 매주 금요일 팀 주간회고에 "이번 주 어땠나" 공유해야 한다. 매번 메일·미팅·Jira 직접 훑어보느라 30분 걸린다.

**대화**

> 정팀장: "이번 주 정리해줘"

Claude가 `/gwh:digest` 실행. 월요일부터 오늘까지의 데이터 자동 집계.

```
📊 Weekly Digest — 2026-W16 (4/13~4/19)

📬 메일
- 🏢 work: 수신 147통 / 송신 42통
  주요 소통: 김팀장 (18통), 박개발자 (12통)
- 👤 personal: 수신 23통 / 송신 2통

📅 미팅
| 계정 | 횟수 | 시간 | 업무시간 대비 |
|------|------|------|--------------|
| 🏢 work | 14회 | 13.5h | 34% ⚠️ 과부하 |

🏃 스프린트 (계정 통합)
- 완료: 6건 (SP 13)
- Bug 2 / Story 3 / Task 1
- 전주 대비: +2건, +5 SP

💡 인사이트
- work 코딩 가용시간: 22h (55%)
- SP 효율: 0.59 SP/h
- ⚠️ 미팅이 34%로 과부하 상태 — 다음 주 미팅 2건 cancellable한지 검토
```

**얻은 것**: 회고 자료가 10초 만에 만들어지고, "미팅 34%"라는 숫자가 뜬 순간 다음 주 캘린더를 바로 정리하는 계기가 됨. 팀에 공유할 때도 감이 아닌 숫자로 대화 가능.

**상황**: 최개발자는 Claude Code 커뮤니티에서 gws-harness 링크를 보고 설치. 회사가 아니라 개인 Gmail만 쓴다. Jira나 Notion은 연결 안 되어 있다.

Claude가 `gws auth login`을 트리거. 브라우저에서 OAuth 로그인 후 돌아옴.

- Label: personal (자동 추론: gmail.com 도메인)

## 시나리오 6: 기여자 — 내 팀 전용 스킬 추가하기

**상황**: 한기여자는 자기 팀의 "월간 보고서 자동 생성" 니즈를 gws-harness에 녹이고 싶다. 기존 스킬을 손대지 않고 새 스킬만 추가하려 한다.

**대화 / 작업**

1. 레포 fork 후 `AGENTS.md` 정독 — 새 스킬 추가 절차 + frontmatter 규칙 파악
2. 2계정 구조를 활용하는 스킬이므로 `skills/shared/references/multi-account.md`의 § 1(계정 인증 검증) § 2(병렬 fetch) 패턴 복사
3. `skills/monthly-report/SKILL.md` 신규 작성 — 기존 `digest` 패턴을 월 단위로 확장
4. PR 생성 → CI가 frontmatter + markdownlint 자동 검증 → 녹색 확인
5. 본인이 로컬에서 `/gwh:monthly-report` 호출 → 정상 동작 확인 후 리뷰 요청

**얻은 것**: 2계정 인프라(`~/.gwh/config.yml`, 병렬 fetch, 파일 권한 0600 자동)를 처음부터 다시 구현할 필요 없음. 스킬 하나만 새로 추가하면 기존 사용자도 즉시 혜택. PR이 머지되면 플러그인 마켓플레이스를 통해 자동 배포.
