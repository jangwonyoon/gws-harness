# 효용성 평가 루브릭

이 문서는 gws-harness 개별 스킬/에이전트의 **효용성(가치)** 을 평가하는 공통 기준이다. 기능 동작 여부가 아니라 "실제 사용자 워크플로우에 가치를 더하는가"를 측정한다.

## 5가지 차원 (각 1-5점)

### D1. 문제 해결력 (Problem Fit) — "선언한 문제를 실제로 푸는가"
- 5 = 스킬이 주장하는 문제를 정확히 해결, 추가 수동 작업 불필요
- 3 = 대체로 해결하나 최소 1번 사용자 보정 필요
- 1 = 출력이 문제와 동떨어져 있음, 수동으로 다시 해야 함

### D2. 신호 대 노이즈 (Signal/Noise) — "출력의 액션 가능한 정보 비율"
- 5 = 출력의 80%+가 즉시 액션 가능, 불필요한 장식/반복 없음
- 3 = 절반은 유용, 절반은 건너뜀
- 1 = 출력 대부분이 템플릿 빈칸/장식/중복

### D3. 컨텍스트 절약 (Context Saving) — "수동 대비 얼마나 효율적인가"
- 5 = 수동으로 했으면 30분+ 걸렸을 작업을 1분 내 완료
- 3 = 5-10분 절약
- 1 = 수동으로 하는 게 더 빠름

### D4. 재사용성 (Composability) — "결과물을 다른 작업이 소비 가능한가"
- 5 = 산출물 파일이 구조화되어 grep/다음 스킬/리포트 작성에 재사용됨
- 3 = 본인 기록용으로만 유용, 외부 소비 어려움
- 1 = 출력이 휘발성, 기록으로 남길 가치 없음

### D5. 채택 확률 (Adoption) — "평가자가 본인 워크플로우에 통합하겠는가"
- 5 = 내일부터 매일/매주 쓰겠다
- 3 = 가끔 쓸 것 같다
- 1 = 안 쓸 것 같다

## 종합 점수

- **25점 (5×5)**: 필수 도구
- **20-24**: 강력 추천
- **15-19**: 유용하나 특정 상황용
- **10-14**: 개선 필요
- **<10**: 스킬 재설계 필요

## 평가 시 주의사항

1. **실제 데이터 필수** — 빈 Gmail/Calendar로 평가하지 말 것. 현실적인 데이터 부하(미읽음 50+통, 미팅 5+개)에서 평가해야 함.
2. **비교 기준 명시** — D3(컨텍스트 절약)은 "수동으로 동일 작업 수행 시 예상 시간" 대비 측정.
3. **채택 판단에 솔직할 것** — D5는 "좋아 보인다"가 아니라 "내일 실제로 쓸까"로 판단.
4. **산출물 검증** — D4는 `~/.gwh/` 파일을 실제로 열어보고 다음 스킬에 재활용 가능한지 확인.
