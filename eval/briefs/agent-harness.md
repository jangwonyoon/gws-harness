# Eval Brief: `gws-harness` AGENT.md (하네스 오케스트레이터)

## 에이전트 목적 (주장)
자연어 요청을 받아 적절한 `/gwh:*` 스킬로 라우팅하거나, gws CLI를 직접 조합하여 처리한다. **코어(Gmail+Calendar)와 확장(Jira/Notion)을 런타임 감지**하여 graceful degradation 제공.

## 해결하겠다는 문제
- 개별 스킬 이름을 외우지 않고 "오늘 뭐해", "메일 정리해줘" 같은 자연어로 접근
- gws CLI 직접 호출보다 의도 표현이 쉬움
- 확장 기능(Jira/Notion) 감지가 자동화되어 사용자 설정 최소화

## 평가 시나리오

### S1. 자연어 라우팅
다음 요청들을 각각 입력하고 **올바른 스킬로 분기**하는지 확인:

| 입력 | 예상 라우팅 |
|------|-----------|
| "오늘 메일 정리해줘" | `/gwh:triage` |
| "오늘 뭐해?" | `/gwh:brief` |
| "이번 주 계획 짜줘" | `/gwh:cal-plan` |
| "이 메일 티켓으로 만들어" | `/gwh:mail-to-ticket` |
| "이번 주 회고 정리" | `/gwh:digest` |
| "회의록 메일 보내줘" | gws CLI 직접 (`gws gmail +send`) |

### S2. 라우팅 모호성 처리
다음 애매한 요청에 대한 반응:
- "메일 확인" → triage인가 brief인가? 명확화 질문 하는가?
- "일정 잡아줘" → cal-plan(배치)인가 gws calendar +insert(단건)인가?

### S3. 사전 확인 (Step 0)
1. 첫 호출 시 `gws auth status` 확인이 일어나는가
2. 인증 실패 시 안내 메시지가 명확한가
3. 확장 감지(Jira/Notion MCP) 결과 보고가 제공되는가 (`✅ Jira 연결됨`)

### S4. 직접 처리 경로
라우팅 테이블에 없는 요청:
- "내일 10시 미팅 잡아줘 @ 홍길동" → `gws calendar +insert` 직접 호출 + 승인 게이트
- "김과장에게 메일 보내줘" → `gws gmail +send` 조합

### S5. 코어 vs 확장 분기
1. Jira MCP 미연결 상태에서 "오늘 뭐해" 호출
2. brief 스킬로 라우팅되되 Jira 섹션은 생략
3. 사용자에게 "Jira 미연결로 스프린트 섹션은 생략됩니다" 안내

### S6. 에러 처리
- gws 명령 실패 (종료코드 1-5)에 대한 각각의 안내 품질
- 특히 2번(인증 에러) 시 `gws auth login` 안내가 즉시 제공되는가

## 측정해야 할 지표

| 지표 | 측정 방법 |
|------|----------|
| 라우팅 정확도 | S1의 6개 요청 중 올바르게 분기된 수 |
| 모호성 처리 | S2에서 명확화 질문 vs 추측 실행 |
| 확장 감지 신뢰성 | S3에서 Jira/Notion 상태 보고 정확도 |
| 오남용 방지 | 쓰기 작업에서 승인 게이트 누락 없는지 |

## 루브릭 적용 가이드

- **D1 문제 해결력**: 개별 `/gwh:*` 이름을 외우지 않아도 되는 가치가 실현되는가?
- **D2 신호/노이즈**: 사전 확인/상태 보고가 도움 vs 귀찮음
- **D3 컨텍스트 절약**: "메일 정리해줘" vs 정확한 커맨드 기억 부담
- **D4 재사용성**: 에이전트 수준에서 만든 컨텍스트(감지 결과 등)가 하위 스킬에 전달되는가
- **D5 채택 확률**: `/gwh:*` 직접 호출 vs 자연어 라우팅 중 무엇을 실제로 쓸 것인가?

## 관찰 포인트

- 자연어 라우팅이 **정확한데 느리다** vs **빠른데 틀린다** 중 어느 쪽이 자주 발생하는가
- 하네스 추상화가 개별 스킬 대비 **가치가 있는가** (단순 wrapper면 불필요)
- Step 0 사전 확인이 매 호출마다 반복되면 **귀찮음** 발생 가능 — 세션 단위 캐싱 필요?
- 직접 처리(라우팅 외) 경로가 실제로 쓰이는가 vs 거의 스킬로만 가는가

## 메타 평가 질문

하네스 에이전트 자체가 **얇은 wrapper**에 머무르지 않고 **오케스트레이션 가치**를 제공하는가? 아니면 개별 `/gwh:*` 스킬만으로도 충분하고 AGENT.md는 과잉인가?
