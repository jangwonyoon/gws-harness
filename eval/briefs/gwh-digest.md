# Eval Brief: `/gwh:digest`

## 스킬 목적 (주장)
이번 주 **메일 송수신 + 캘린더 참석 + (optional) Jira 완료 티켓**을 집계하여 주간 리포트 생성. 미팅 과부하(>30%), 소통 편중 등 인사이트 자동 도출.

## 해결하겠다는 문제
- 금요일 회고 시 "이번 주 뭐 했지" 파악에 수동 집계 필요
- 미팅 시간 비중, 완료 티켓 같은 메타 지표 추적 어려움

## 평가 시나리오

### S1. 주 후반 실행 (Happy Path)
1. 목/금 실행 (한 주의 데이터가 충분한 상태)
2. `/gwh:digest` 실행
3. 3섹션 확인:
   - 📬 메일 (수신/송신 카운트, 주요 발신자)
   - 📅 미팅 (횟수/시간/비중)
   - 🏃 Jira (완료 건수, Bug/Story/Task 분포)

### S2. 주 초 실행 (Edge Case)
1. 월/화에 실행 (데이터 부족)
2. "데이터가 적습니다. 지난주 데이터 포함할까요?" 제안이 나오는가
3. 폴백 동작 확인

### S3. Jira 미연결 폴백
1. Atlassian MCP 연결 해제 후 실행
2. 메일+캘린더 2섹션만 출력되는지
3. Jira 섹션이 생략되지만 다른 인사이트(미팅 비중 등)는 여전히 나오는가

### S4. 인사이트 품질 검증
1. 미팅 비중이 30% 초과인 주에 실행 → 과부하 경고 나오는가?
2. 특정 사람과 메일이 50% 이상 집중된 경우 → 편중 경고 나오는가?
3. 코딩 가용시간 계산이 현실적인가?

### S5. 전주 대비 변화
1. 전주에도 `/gwh:digest`를 실행했던 경우 (`~/.gwh/digest-{YYYY}-W{WW-1}.md` 존재)
2. "전주 대비 미팅 +N회, SP ±M" 같은 비교 데이터가 나오는가

### S6. 산출물 재사용
1. `~/.gwh/digest-{this-week}.md` 파일 확인
2. 팀 회고/스탠드업에 복붙해서 쓸 만한 수준인가
3. velog/블로그 주간 회고 글 소재로 가공 가능한가

## 측정해야 할 지표

| 지표 | 측정 방법 |
|------|----------|
| 메일 카운트 정확도 | 출력 숫자 vs Gmail "이 주" 필터 수동 확인 |
| 미팅 시간 정확도 | 출력 시간 vs Calendar 수동 집계 |
| 인사이트 유효성 | "과부하 경고" 등이 실제 체감과 일치하는가 |
| 송신 0건 케이스 | 실제로 메일을 안 보낸 주에 어떻게 표현되는가 |

## 루브릭 적용 가이드

- **D1 문제 해결력**: 회고 자료를 빠르게 만들어주는가?
- **D2 신호/노이즈**: 인사이트가 "당연한 말" vs "몰랐던 패턴"
- **D3 컨텍스트 절약**: 수동 집계 20-30분 vs 스킬 사용 시
- **D4 재사용성**: 리포트가 velog/팀 공유에 즉시 사용 가능?
- **D5 채택 확률**: 매주 금요일에 자동 실행할 루틴을 만들 것인가?

## 관찰 포인트

- 미팅 비중 30% 임계가 본인 팀 문화와 맞는가 (조직마다 다름)
- "주요 소통" Top 3가 실제 체감과 맞는가
- 메일 송신 0건처럼 예외 케이스가 왜곡된 인사이트를 만들지 않는지
- 주간 회고 글 쓰는 사람이라면 이걸로 초안이 바로 나오는가

## 읽기 전용 스킬 (안전)

이 스킬은 쓰기 작업 없음. Gmail/Calendar/Jira 모두 read-only로만 접근. 편하게 여러 번 실행 가능.
